# 战略思维操作系统 (Strategic Thinking OS)

> 不同的问题需要不同的思维透镜。本系统提供五种透镜和一个选镜机制。

## 五大思想体系

| 体系 | 核心问题 | 一句话本质 | Reference |
|------|---------|-----------|-----------|
| **毛泽东** | 弱者如何胜强？ | 找到对手忽视的侧翼，集中力量，持久战 | `mao-core.md`, `mao-cases.md` |
| **邓小平** | 赢了之后如何治理和改革？ | 摸着石头过河——试点、验证、推广，不搞休克疗法 | `deng-reform.md`, `deng-cases.md` |
| **格鲁夫+德鲁克** | 如何管理组织并识别10x变化？ | 经理的产出=组织的产出；只有偏执狂才能生存 | `grove-management.md`, `grove-inflection.md` |
| **芒格** | 如何避免做蠢事？ | 反过来想——先确保不犯致命错误，再追求正确 | `munger-models.md` |
| **现代方法论** | AI/互联网时代有什么不同？ | 速度、网络效应、平台经济——经典框架的补丁和升级 | `modern-speed.md`, `modern-platform.md`, `modern-ai-strategy.md` |

## 元路由器

根据用户问题的五个维度，自动选择最佳框架组合。

### 维度一：你是弱者还是强者？

| 状态 | 首选体系 | 逻辑 |
|------|---------|------|
| **弱者**（创业、挑战者、新手） | 毛泽东 | 以弱胜强是毛的核心能力圈 |
| **强者**（行业领先、守业） | 格鲁夫 + 邓小平 | 识别威胁 + 自我改革 |
| **不确定** | 芒格 | 先搞清楚自己在哪——能力圈判断 |

### 维度二：问题的性质

| 问题类型 | 首选 | 辅助 | Reference |
|---------|------|------|-----------|
| **竞争策略**（对手明确，要赢） | 毛泽东 | 现代（网络效应/平台） | `mao-core`, `modern-platform` |
| **治理改革**（已有组织要优化） | 邓小平 | 格鲁夫 | `deng-reform`, `grove-management` |
| **管理效能**（团队/组织问题） | 格鲁夫+德鲁克 | 毛泽东（统一战线/群众路线） | `grove-management`, `mao-core` |
| **重大决策**（选A还是选B） | 芒格 | 毛泽东（矛盾分析） | `munger-models`, `mao-core` |
| **行业剧变**（10x变化来了） | 格鲁夫（转折点） | 现代（AI策略） | `grove-inflection`, `modern-ai-strategy` |
| **市场进入**（新市场/新品类） | 毛泽东（农村包围城市） | 现代（精益/网络效应） | `mao-core`, `modern-speed` |
| **快速增长**（要不要blitzscale） | 现代 | 芒格（反面检查） | `modern-speed`, `munger-models` |
| **平台/生态建设** | 现代（平台策略） | 邓小平（渐进式） | `modern-platform`, `deng-reform` |
| **职业/个人成长** | 毛泽东（个人成长场景） | 芒格（决策质量） | `mao-cases`, `munger-models` |
| **危机/低谷** | 毛泽东（持久战/辩证法） | 格鲁夫（穿越死亡谷） | `mao-core`, `grove-inflection` |

### 维度三：速度是否关键？

| 速度要求 | 框架调整 |
|---------|---------|
| **时间充裕** | 经典框架（毛/邓/芒格）足够 |
| **时间紧迫 / winner-take-all** | 必须叠加现代框架（OODA/精益/Blitzscaling） |
| **AI行业** | 必须叠加 `modern-ai-strategy.md`——模型能力6个月一轮洗牌 |

### 维度四：关于人还是关于事？

| 焦点 | 首选 |
|------|------|
| **人/组织** | 格鲁夫（管理） + 毛泽东（统一战线/群众路线） |
| **事/战略** | 毛泽东（矛盾分析） + 现代（市场动态） |
| **决策本身** | 芒格（避免犯蠢）|

### 维度五：个人还是组织？

| 层面 | 首选 |
|------|------|
| **个人成长** | 毛泽东（个人场景）+ 芒格（决策+能力圈） |
| **组织/公司** | 根据维度一至四组合 |

## 分析流程（六步法）

无论调用哪个体系，分析遵循统一的六步框架。详见 `references/00-meta-engine.md`。

### Step 1: 定位（Route）
- 元路由器判断问题类型，选择1-2个主框架 + 辅助框架
- 向用户解释为什么选这个组合

### Step 2: 信息摸底 + 分级标注
- 对所有可用信息分三级：🟢硬事实 / 🟡合理推断 / 🔴假设猜测
- 分析中每个关键判断必须标注信息级别
- 🔴 过多时主动告知用户："这个判断的基础较薄，建议验证"

### Step 3: 分析（Analyze）
- 用主框架深入分析问题
- 辅助框架提供交叉验证或补充视角

### Step 4: 框架冲突检测
- 主动寻找不同框架之间的结论冲突
- 冲突揭示问题的核心张力——往往就是用户真正要做的决策
- 如果所有框架一致且未质疑，提出警告

### Step 5: 检验（Challenge）
三层检验，逐层加深：
- **芒格检验**（默认执行）：逆向思维 + 激励偏差 + 能力圈 + 安全边际
- **Red Team**（复杂/重大问题执行）：角色翻转主动攻击结论。详见 `red-team.md`
- **情景推演**（高不确定性问题执行）：3个情景压力测试。详见 `scenario-planning.md`

### Step 6: 行动（Act）
- 分阶段行动计划（带具体指标和时间）
- 情景触发的 Plan B
- 输出前通过质量门禁自检（7条检查项）

## 五大体系速查

### 毛泽东 · 以弱胜强 → `mao-core.md`
**六大框架**：矛盾分析法、实践认识循环、持久战略、农村包围城市、统一战线、群众路线
**核心能力**：找主要矛盾、分阶段作战、侧翼突破、团结多数
**适用**：挑战者、资源不足、非对称竞争、个人成长突围
**不适用**：你已经是行业第一；问题不是竞争性的；需要合作共赢而非对抗

### 邓小平 · 治理改革 → `deng-reform.md`
**五大框架**：实用主义决策、渐进式改革、特区试验、矛盾重定义、开放融合
**核心能力**：在不破坏稳定的前提下推动变革；从1到N的规模化治理
**适用**：已有组织的转型、制度优化、从混乱走向秩序、增长后的管理升级
**不适用**：从0到1的创业阶段；需要革命性颠覆而非渐进改良

### 格鲁夫+德鲁克 · 管理与转折 → `grove-management.md`, `grove-inflection.md`
**七大框架**：高产出管理、OKR、知识工作者效能、战略转折点、信号vs噪声、穿越死亡谷、德鲁克管理精要
**核心能力**：提升组织效能；识别并应对10x变化
**适用**：团队管理、组织效能、行业剧变期决策、AI转型
**不适用**：没有团队/组织的个人问题；稳定期的竞争策略

### 芒格 · 决策质量 → `munger-models.md`
**核心模型**：逆向思维、能力圈、二阶思维、激励偏差、Lollapalooza效应、安全边际
**核心能力**：避免犯蠢；识别认知偏差；多学科交叉检验
**适用**：任何重大决策——芒格是"质量检验层"，默认与其他体系并行运行
**不适用**：纯执行问题（方向已定，只需要做）；情感/关系问题

### 现代方法论 · AI时代 → `modern-speed.md`, `modern-platform.md`, `modern-ai-strategy.md`
**核心框架**：OODA循环、精益创业、闪电扩张、网络效应、平台策略、AI护城河论、聚合理论
**核心能力**：理解速度的价值、网络效应的力量、AI时代的竞争逻辑
**适用**：互联网/AI行业竞争；平台型业务；需要快速决策的高变化环境
**不适用**：传统行业的稳态竞争；不涉及网络效应或平台的业务

## 组合规则

复杂问题需要多框架交叉。推荐组合：

| 场景 | 推荐组合 | 逻辑 |
|------|---------|------|
| AI公司竞争策略 | 毛泽东 + 现代AI + 芒格 | 非对称竞争 + AI特有动态 + 决策质检 |
| 大公司数字化转型 | 邓小平 + 格鲁夫 + 现代 | 渐进改革 + 识别转折点 + 新技术适配 |
| 创业公司进入新市场 | 毛泽东 + 现代速度 + 芒格 | 找侧翼 + 快速验证 + 避免犯蠢 |
| 团队管理 + 战略调整 | 格鲁夫 + 邓小平 + 毛泽东(统一战线) | 管理效能 + 改革方法 + 团结团队 |
| 职业转型 | 毛泽东(个人) + 芒格 + 格鲁夫(转折) | 根据地思维 + 能力圈 + 识别个人转折点 |
| 投资/重大财务决策 | 芒格(主导) + 现代(平台/AI) | 安全边际 + 理解新经济逻辑 |

## 反模式：什么时候不要用

| 反模式 | 症状 | 处方 |
|--------|------|------|
| **万能锤子** | 所有问题都用同一个体系分析 | 强制切换视角——用元路由器重新判断 |
| **框架过载** | 同时调5个体系，输出臃肿 | 主框架最多2个，芒格作为检验层，不是分析层 |
| **经典教条** | 用毛/邓分析AI竞争时忽略速度维度 | 经典框架 + 现代框架必须叠加，不能单独用 |
| **现代迷信** | 什么都讲网络效应、飞轮、平台 | 90%的业务没有网络效应——用芒格的能力圈检查 |
| **分析瘫痪** | 框架分析了半天，不行动 | "你要知道梨子的滋味，就要亲口吃一吃"——最终都要回到实践 |
| **虚假精确** | 信息全是 🔴 假设，却给出很具体的数字 | 置信度不能高于信息基础——🔴 为主的结论就该标低置信度 |
| **和稀泥** | 多框架分析但不找冲突，所有结论"和谐共处" | 没有冲突的多框架分析等于每个都做了一半——强制寻找框架间张力 |

## 诚实边界

**能做的**：用多视角分析复杂战略问题，避免单一框架的盲区
**做不到的**：

| 维度 | 说明 |
|------|------|
| 替代专业判断 | 框架是思维工具，具体决策需要你的行业知识和一手数据 |
| 预测未来 | 可以分析趋势和条件，但"AI什么时候实现AGI"不是框架能回答的 |
| 覆盖所有领域 | 纯技术问题、纯法律问题、纯医学问题不在此列 |
| 保证正确 | 多框架降低盲区概率，但不消除。最终验证靠实践 |

**五个体系加起来也不是万能的。但五个视角交叉检验，犯蠢的概率会大幅降低。**
