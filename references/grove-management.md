# 格鲁夫+德鲁克 · 管理效能

> "一个经理的产出 = 他的组织的产出 + 他影响力所及的相邻组织的产出。" ——Andy Grove《高产出管理》
> "管理的本质不是知道，而是做到。" ——Peter Drucker

---

## 核心定位

格鲁夫解决"如何让组织高效运转"和"如何识别致命变化"。德鲁克提供管理的底层原理。

---

## 格鲁夫 · 高产出管理

### 1. 经理的产出公式

**核心**：经理不是靠自己干活产生价值，而是通过提升团队产出产生价值。

```
经理的产出 = 组织产出 + 影响力波及的相邻组织产出
```

**推论**：
- 你亲自写代码1小时 vs 帮10个工程师每人提效10% → 后者杠杆更高
- 好的1:1会议可能是你一周中杠杆最高的30分钟
- 审阅关键设计文档可能比自己写更有价值

**操作**：每周问自己："我这周做的事中，哪些有高杠杆？哪些我不该亲自做？"

### 2. 管理杠杆

| 活动 | 杠杆率 | 举例 |
|------|--------|------|
| **高杠杆** | 一个行动影响很多人或长期影响 | 设计好流程、培训团队、做关键决策、清除障碍 |
| **中杠杆** | 一对一影响 | 1:1 coaching、代码review、面试 |
| **低杠杆** | 自己做、或做可委托的事 | 亲自写不重要的文档、参加不需要你的会议 |
| **负杠杆** | 你的介入反而降低产出 | 微管理、在不懂的领域指手画脚、拖延决策 |

**操作**：把每天的活动标记杠杆率。高杠杆活动应该占50%+。如果低杠杆占多数，你在做IC的活，不是经理的活。

### 3. 会议作为管理媒介

Grove 认为会议不是浪费时间——**会议是经理的工作本身**。

| 会议类型 | 目的 | 频率 | 关键规则 |
|---------|------|------|---------|
| **1:1** | 上下级信息交换+coaching | 每周/双周 | 下属定议程，经理听和问。不是汇报会 |
| **团队周会** | 信息同步+跨职能协调 | 每周 | 有议程，有结论，有follow-up。没结论的会是浪费 |
| **决策会** | 解决重大分歧 | 按需 | 事前发材料，会上辩论，会后一人拍板 |

### 4. OKR（Objectives and Key Results）

Grove在Intel发明，后被Google大规模采用。

**结构**：
- **Objective**：你想达成什么？（定性，有方向感）
- **Key Results**：怎么知道达成了？（定量，可衡量）

**操作规则**：
- 每个层级3-5个O，每个O有2-4个KR
- KR必须可量化——"提升用户体验"不是KR，"NPS从40提升到55"是
- 60-70%完成率是健康的——100%完成说明目标太保守
- 向上对齐：你的O应该支撑上级的某个KR

## 德鲁克 · 管理精要

### 5. 知识工作者效能

> "20世纪最重要的管理贡献是将体力工作者的生产率提高了50倍。21世纪最重要的管理贡献将是提高知识工作者的生产率。"

**核心洞察**：知识工作者的产出无法像流水线那样度量。管理知识工作者需要不同的方法。

**操作原则**：
1. **定义"产出"**——知识工作者首先要回答：我被期望的成果是什么？（不是"做什么工作"而是"产出什么结果"）
2. **自主管理**——知识工作者必须自己管理自己的时间、优先级、质量
3. **持续学习**——知识工作就是持续学习。给时间、给资源、给犯错空间
4. **优势导向**——安排人做他擅长的事，不要试图把短板补成长板

### 6. 战略性放弃

> "最重要的决策不是做什么，而是不做什么。"

**操作**：
- 每年/每季度做一次"系统性放弃"审查
- 问："如果我们今天没有在做这件事，我们会开始做吗？"
- 如果答案是"不会"→停掉它，把资源释放给更重要的事
- 组织里90%的"资源不足"问题，本质是"不愿意停掉不该做的事"

### 7. 创新与企业家精神

**创新七大来源**（按可靠性排序）：
1. 意外事件——意外成功比意外失败更值得分析
2. 不协调——"应该如此"和"实际如此"之间的差距
3. 流程需要——某个流程中的薄弱环节
4. 行业/市场结构变化
5. 人口结构变化
6. 认知变化——社会态度和价值观的转变
7. 新知识——技术突破（最不可靠，周期最长）

## 格鲁夫+德鲁克的适用边界

| 擅长 | 不擅长 |
|------|--------|
| 组织效能提升 | 从0到1的竞争策略 → 用毛泽东 |
| 经理/管理者的能力提升 | 个人贡献者的技术成长 |
| 结构化管理流程 | 混乱初创期的生存问题 → 用毛泽东+精益 |
| 识别并应对10x变化 | 稳定环境中的渐进优化 → 用邓小平 |
| 知识工作者团队 | 体力劳动为主的组织 |
